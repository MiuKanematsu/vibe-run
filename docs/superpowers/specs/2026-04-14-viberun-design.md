# VibeRun — 設計ドキュメント

**作成日:** 2026-04-14  
**フェーズ:** MVP  
**プラットフォーム:** iOS（Expo / React Native）

---

## 1. コンセプト

「一人でカフェに行くより、もっと楽しい週末を。」

ランニングを「記録」するためではなく、今この瞬間に動いている「コミュニティの熱量」を可視化し、20代が最もラントリップや交流をカジュアルに楽しめるプラットフォーム。

### 解決するペインポイント
- **不可視性:** インフルエンサーやコミュニティがどこで走っているか、リアルタイムで把握する手段がない
- **心理的障壁:** ガチ勢への恐怖心、既存コミュニティの内輪感
- **手段の欠如:** Instagramは報告には向くが「今から一緒に」という合流の導線が弱い

---

## 2. MVPスコープ

### 含む機能
| 機能 | 説明 |
|---|---|
| 認証 + Vibeプロフィール | Apple/Googleログイン、名前・アイコン・Vibeタグ |
| ライブランマップ | セッションピンの作成・参加・リアルタイム更新 |
| ストーリーズフィード | 写真＋走行データオーバーレイ、24時間で消える投稿 |
| カフェ目的地設定 | セッション作成時にカフェを設定、走後にサジェスト |

### MVPから除外
| 機能 | 理由 |
|---|---|
| ヒートマップ | ユーザー数が少ないと機能しない |
| カフェ割引・特典 | 店舗との契約が必要（ビジネス開発フェーズ） |
| 「Nice Vibe!」評価 | ユーザー数が必要 |
| Instagram自動投稿 | 不要（要件から除外） |

## 4. データモデル

### `users`
```
{
  id: string,
  name: string,
  avatar: string,          // Firebase Storage URL
  vibeTags: string[],      // 例: ["カフェ好き", "喋りやすい", "お洒落"]
  createdAt: timestamp
}
```

### `sessions`
```
{
  id: string,
  creatorId: string,
  title: string,
  location: GeoPoint,
  locationName: string,    // 例: "代々木公園"
  startTime: timestamp,
  pace: string,            // 例: "キロ7分（ゆっくり）"
  cafeDestination: string, // カフェ名（任意）
  cafeLocation: GeoPoint,  // カフェ位置（任意）
  participants: string[],  // userId[]
  status: "open" | "active" | "finished",
  createdAt: timestamp
}
```

### `stories`
```
{
  id: string,
  userId: string,
  mediaUrl: string,        // Firebase Storage URL
  overlayData: {
    distance: number,      // km
    duration: number,      // 秒
    routePolyline: string  // エンコード済みルート
  },
  location: GeoPoint,
  sessionId: string,       // 紐付けセッション（任意）
  expiresAt: timestamp,    // 作成から24時間後
  createdAt: timestamp
}
```

## 5. 画面構成

### ボトムタブナビゲーション
```
[ マップ ] [ フィード ] [ 投稿+ ] [ プロフィール ]
```

### 画面フロー
```
起動
 └─ ログイン画面（Apple/Google）
     └─ Vibeプロフィール設定（初回のみ）
         └─ メイン画面へ

【マップタブ】
 ├─ 近くのセッションピンを表示
 ├─ ピンタップ → セッション詳細
 │    └─ ワンタップ参加
 └─ ＋ボタン → セッション作成
      ├─ 場所・時間・ペース設定
      └─ カフェ目的地設定（任意）

【フィードタブ】
 ├─ フォロー中 / 近く の切り替え
 └─ ストーリーズ一覧（24時間で消える）

【投稿タブ】（中央の＋）
 ├─ 写真・動画撮影
 └─ 走行データオーバーレイを合成して投稿

【プロフィールタブ】
 ├─ Vibeタグ編集
 └─ 過去の参加履歴
```

## 6. エラーハンドリング

| ケース | 対処 |
|---|---|
| 位置情報の許可拒否 | 許可を促すモーダル表示、マップは閲覧のみ可能 |
| ネットワーク切断 | トースト通知、オフライン時はキャッシュ表示 |
| ストーリー投稿失敗 | リトライボタンを表示 |

---

## 7. テスト方針（MVP）

- **手動テスト中心** — シミュレーターと実機で主要フローを確認
- **Firebase Emulator** — ローカルで Firestore・Auth を動かして開発
- ユニットテストはMVP後に追加

---

## 8. UXフロー（ユーザー体験）

1. **発見** — 週末の朝、アプリを開くと近所の公園にピンがいくつか立っている
2. **選択** — 自分のVibeに合う募集（ペース・カフェ先）を選ぶ
3. **合流** — 集合場所へ行き、アプリで「参加」をワンタップ
4. **体験** — 30〜40分お喋りしながら走り、目当てのカフェへ
5. **シェア** — カフェでの一枚をストーリーズにアップ（走行データが自動合成）
6. **完了** — 「またどこかの道で！」と解散
